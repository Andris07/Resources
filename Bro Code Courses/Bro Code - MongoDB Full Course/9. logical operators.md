# MongoDB Logical Operators

# Basic Syntax

### `db.<collection name>.find({$<logical operator>: [{<fieldKey>: <fieldValue>}, {<fieldKey>: {$<comparison operator>: <fieldValue>}}]})`

Logical operators combine multiple conditions in a query.

---

# `$and` — All Conditions Must Be True

Returns documents only if every condition matches.

Example:

```
db.students.find({
    $and: [
        {fullTime: true},
        {age: {$lte: 22}}
    ]
})
```

This command finds students who:

* are full-time students
* and are `22` years old or younger

---

# `$or` — At Least One Condition Must Be True

Returns documents if at least one condition matches.

Example:

```
db.students.find({
    $or: [
        {fullTime: true},
        {age: {$lte: 22}}
    ]
})
```

This command finds students who:

* are full-time students
* or are `22` years old or younger

---

# `$nor` — None of the Conditions Can Be True

Returns documents only if all conditions are false.

Example:

```
db.students.find({
    $nor: [
        {fullTime: true},
        {age: {$lte: 22}}
    ]
})
```

This command finds students who:

* are not full-time students
* and are older than `22`

---

# `$not` — Reverses a Condition

Negates a comparison operator.

Example:

```
db.students.find({
    age: {$not: {$gte: 30}}
})
```

This command finds students whose age is NOT greater than or equal to `30`.

In other words:

* finds students younger than `30`

---

# Important Notes

## `$and`

All conditions must match.

---

## `$or`

At least one condition must match.

---

## `$nor`

None of the conditions can match.

---

## `$not`

Reverses the result of a comparison condition.
