# MongoDB Indexing Commands

# Create an Index

### `db.<collection name>.createIndex({<fieldKey>: <sortType>})`

Creates an index for a field.

Indexes improve search and query performance.

* `1` = ascending index
* `-1` = descending index

Example:

```
db.students.createIndex({name: 1})
```

This command creates an ascending index for the `name` field.

---

# View Existing Indexes

### `db.<collection name>.getIndexes()`

Displays all indexes in the collection.

Example:

```
db.students.getIndexes()
```

MongoDB automatically creates a default `_id` index for every collection.

---

# Delete an Index

### `db.<collection name>.dropIndex("<indexName>")`

Deletes a specific index.

Example:

```
db.students.dropIndex("name_1")
```

This command removes the `name_1` index.

The index name usually follows this format:

```
<fieldName>_<sortType>
```

Example:

* `name_1`
* `gpa_-1`

---

# View Indexes Again

You can check the remaining indexes after deleting one.

Example:

```
db.students.getIndexes()
```

---

# Important Notes

## Why Use Indexes?

Indexes help MongoDB:

* search faster
* sort data faster
* improve query performance

---

## Default `_id` Index

Every collection automatically has an `_id` index created by MongoDB.
