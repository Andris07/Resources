# Trees

# What is a Tree?

A **tree** is a hierarchical, non-linear data structure consisting of **nodes** connected by **edges**.

Unlike graphs, a tree has the following properties:

- It has exactly **one root node**.
- Every node (except the root) has exactly **one parent**.
- There is exactly **one unique path** between any two nodes.
- A tree contains **no cycles**.

Trees are commonly used to represent hierarchical relationships such as file systems, organization charts, and XML/HTML documents.

Example:

```text
            A
          / | \
         B  C  D
        / \     \
       E   F     G
```

# Basic Tree Terminology

The following diagram will be used to explain the most important tree concepts.

```text
            A
          / | \
         B  C  D
        / \     \
       E   F     G
```

---

## Root Node

The **root node** is the topmost node of the tree.

Every tree has exactly one root.

Example:

```text
        A  ← Root
       / \
      B   C
```

---

## Parent Node

A **parent node** is a node that has one or more children.

Example:

```text
    A
   /
  B

A is the parent of B.
```

---

## Child Node

A **child node** is a node directly connected below another node.

Example:

```text
    A
   /
  B

B is the child of A.
```

---

## Sibling Nodes

**Siblings** are nodes that share the same parent.

Example:

```text
      A
    / | \
   B  C  D
```

Nodes **B**, **C**, and **D** are siblings.

---

## Leaf Node

A **leaf node** (also called an external node) is a node with **no children**.

Example:

```text
      A
     / \
    B   C

B and C are leaf nodes.
```

---

## Branch (Internal) Node

A **branch node** (or internal node) is a node that has at least one child.

Example:

```text
      A
     /
    B
   /
  C
```

Nodes **A** and **B** are branch nodes.

---

## Subtree

A **subtree** is a tree formed by a node and all of its descendants.

Example:

```text
            A
          / | \
         B  C  D
        / \
       E   F
```

The subtree rooted at **B** is:

```text
      B
     / \
    E   F
```

---

## Tree Size

The **size** of a tree is the total number of nodes it contains.

Example:

```text
        A
      / | \
     B  C  D
    / \     \
   E   F     G
```

Tree size = **7 nodes**

---

## Tree Depth

The **depth** of a node is the number of edges from the root to that node.

Example:

```text
        A
       /
      B
     /
    C
```

| Node | Depth |
|------|------:|
| A | 0 |
| B | 1 |
| C | 2 |

---

## Tree Height

The **height** of a node is the number of edges on the longest path from that node to a leaf.

The height of the entire tree is the height of the root.

Example:

```text
        A
       /
      B
     /
    C
```

| Node | Height |
|------|-------:|
| C | 0 |
| B | 1 |
| A | 2 |

# Advantages of Trees

Trees provide several advantages:

- Naturally represent hierarchical data.
- Efficient searching, insertion, and deletion in many tree types.
- Easy organization of parent-child relationships.
- Highly scalable for large datasets.
- Foundation for many advanced data structures.

---

# Disadvantages of Trees

Trees also have some limitations:

- More complex than arrays or linked lists.
- Performance depends on the tree structure.
- Some tree operations require balancing.
- Can consume additional memory due to node references.

---

# Where Are Trees Used?

Trees are widely used throughout computer science.

Common applications include:

- File systems and directory structures.
- Binary Search Trees (BST).
- AVL Trees.
- Red-Black Trees.
- B-Trees and B+ Trees.
- Heaps (Priority Queues).
- XML and HTML document structures.
- Database indexing.
- Compilers and Abstract Syntax Trees (ASTs).
- Decision trees in Machine Learning.

---

# Why Are Trees Important?

Trees are one of the most fundamental data structures in computer science.

They provide an efficient way to organize, search, and manipulate hierarchical data. Many advanced data structures—including Binary Search Trees, AVL Trees, Heaps, B-Trees, and Tries—are built upon the concepts introduced by trees. Understanding basic tree terminology is essential before learning tree traversal algorithms and specialized tree implementations.
