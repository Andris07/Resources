# Recursion

# What is Recursion?

**Recursion** is a programming technique in which a function calls itself to solve a problem.

Instead of solving the entire problem at once, the function repeatedly solves smaller versions of the same problem until a stopping condition is reached.

Every recursive function consists of two essential parts:

* **Base Case** ŌĆō Stops the recursion.
* **Recursive Case** ŌĆō Calls the function again with a smaller or simpler input.

Without a base case, the function would continue calling itself indefinitely, eventually causing a **Stack Overflow**.

---

# How Does Recursion Work?

Each recursive function call is placed onto the **Call Stack**.

When the base case is reached, the function calls begin returning one by one in reverse order until the original call finishes.

Each recursive call waits until the next one finishes.

# Base Case vs Recursive Case

Every recursive function should follow the same structure.

Example:

```text
Function()

if stopping condition
    return

otherwise
    call Function(smaller problem)
```

The base case prevents infinite recursion, while the recursive case gradually moves toward the base case.

# Time Complexity

The time complexity of recursion depends entirely on the algorithm being implemented.

Examples:

| Algorithm                 | Time Complexity |
| ------------------------- | --------------- |
| Walk Example              | O(n)            |
| Recursive Factorial       | O(n)            |
| Recursive Power           | O(n)            |
| Binary Search (Recursive) | O(log n)        |
| Merge Sort                | O(n log n)      |

Recursion itself does **not** determine the complexity ŌĆö the underlying algorithm does.

---

# Space Complexity

Recursive functions require additional memory because every function call is stored on the **Call Stack**.

Examples:

| Algorithm           | Space Complexity |
| ------------------- | ---------------- |
| Walk Example        | O(n)             |
| Recursive Factorial | O(n)             |
| Recursive Power     | O(n)             |

An iterative solution to the same problem may require only **O(1)** additional memory.

# Advantages

Recursion provides several advantages:

* Produces elegant and concise code.
* Closely matches many mathematical definitions.
* Simplifies many divide-and-conquer algorithms.
* Naturally solves tree and graph traversal problems.
* Often easier to understand than complex iterative solutions.

---

# Disadvantages

Recursion also has several limitations:

* Uses additional stack memory.
* Function call overhead can reduce performance.
* May cause a **Stack Overflow** if the recursion is too deep.
* Sometimes more difficult to debug than iterative solutions.

---

# When Should You Use Recursion?

Recursion is an excellent choice when:

* The problem can naturally be divided into smaller subproblems.
* Working with trees or graphs.
* Implementing divide-and-conquer algorithms.
* Traversing hierarchical structures.

Typical examples include:

* Factorial calculation.
* Fibonacci sequence.
* Binary Search.
* Merge Sort.
* Quick Sort.
* Tree traversal.
* Graph traversal (DFS).

# Why Is Recursion Important?

Recursion is one of the fundamental concepts in computer science.

Many advanced algorithms and data structures rely on recursive thinking, including tree traversals, graph algorithms, divide-and-conquer techniques, and dynamic programming.

Understanding recursion helps develop a problem-solving mindset where large problems are broken down into smaller, more manageable subproblems.
