# Linear Search

# What is Linear Search?

**Linear Search** is one of the simplest searching algorithms.

It searches for a target value by checking each element in a collection one by one until the value is found or the end of the collection is reached.

Linear Search does **not require the data to be sorted**, making it useful for searching unsorted collections.

---

# How Does Linear Search Work?

The algorithm starts at the beginning of the collection.

Each element is compared with the target value.

* If the current element matches the target, the search stops and returns its index.
* Otherwise, the algorithm continues with the next element.
* If the end of the collection is reached without finding the value, the algorithm returns a value indicating that the search failed (commonly `-1`).

Example:

```text
Array:

[9] [1] [8] [2] [7] [3] [6] [4] [5]
 ↑
 Check 9 ❌

[9] [1] [8] [2] [7] [3] [6] [4] [5]
     ↑
   Check 1 ✅

Found at index 1
```

---

# Pseudocode

```text
LinearSearch(array, target)

for each element in the array
    if element equals target
        return its index

return -1
```

# Time Complexity

The running time depends on where the target value is located.

| Case         | Time Complexity |
| ------------ | --------------- |
| Best Case    | O(1)            |
| Average Case | O(n)            |
| Worst Case   | O(n)            |

### Best Case

The target is the first element.

Example:

```text
[5] 8 2 9 1
 ↑
Found immediately
```

---

### Worst Case

The target is the last element or does not exist.

Example:

```text
5 8 2 9 [1]
         ↑
Checked every element
```

---

# Space Complexity

Linear Search uses only a few variables regardless of the input size.

Space Complexity:

```text
O(1)
```

No additional data structures are required.

---

# Advantages

Linear Search provides several advantages:

* Very simple to understand and implement.
* Works on both sorted and unsorted collections.
* Requires no preprocessing or sorting.
* Uses constant extra memory **O(1)**.
* Suitable for small datasets.

---

# Disadvantages

Linear Search also has several limitations:

* Slow for large collections.
* May need to examine every element.
* Much less efficient than Binary Search on sorted data.
* Performance decreases linearly as the input size grows.

---

# When Should You Use Linear Search?

Linear Search is a good choice when:

* The collection is small.
* The data is unsorted.
* Simplicity is more important than performance.
* The search operation is performed infrequently.
* Sorting the data would introduce unnecessary overhead.

Typical examples include:

* Searching through a small list.
* Finding a value in an unsorted array.
* Educational examples for learning searching algorithms.
* Quick one-time searches.

---

# Why Is Linear Search Important?

Linear Search is often the first searching algorithm taught in Data Structures and Algorithms.

Although it is not the fastest search algorithm, it introduces the fundamental concept of sequential searching and serves as the foundation for understanding more advanced algorithms such as **Binary Search**, **Jump Search**, and **Interpolation Search**.
