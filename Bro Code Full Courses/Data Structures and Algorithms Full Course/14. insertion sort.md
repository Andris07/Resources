# Insertion Sort

# What is Insertion Sort?

**Insertion Sort** is a simple comparison-based sorting algorithm.

It builds the final sorted collection **one element at a time** by repeatedly taking the next element and inserting it into its correct position within the already sorted portion of the array.

Unlike Bubble Sort and Selection Sort, Insertion Sort is **adaptive**, meaning it performs very efficiently on partially sorted data.

---

# How Does Insertion Sort Work?

The algorithm assumes that the first element is already sorted.

For each remaining element:

1. Store the current element.
2. Compare it with the elements before it.
3. Shift larger elements one position to the right.
4. Insert the current element into its correct position.

Example:

```text
Initial Array

[9] [1] [8] [2]

Sorted | Unsorted
            ↑

Take 1

9 > 1

Shift 9

↓

[1] [9] [8] [2]

Sorted Portion

[1] [9]
```

Each iteration expands the sorted portion by one element.

---

# Visualization

```text
Pass 1

9 | 1 8 2

Pass 2

1 9 | 8 2

Pass 3

↓

```

The vertical bar (`|`) separates the sorted and unsorted portions.

---

# Pseudocode

```text
InsertionSort(array)

for i from 1 to length - 1

    current = array[i]

    j = i - 1

    while j >= 0 and array[j] > current

        array[j + 1] = array[j]

        j = j - 1

    array[j + 1] = current
```

# Time Complexity

| Case         | Time Complexity |
| ------------ | --------------- |
| Best Case    | O(n)            |
| Average Case | O(n²)           |
| Worst Case   | O(n²)           |

When the collection is already sorted, Insertion Sort only performs one comparison per element, making it much faster than Bubble Sort and Selection Sort.

---

# Space Complexity

```text
O(1)
```

Insertion Sort is an **in-place** sorting algorithm because it only uses a small number of additional variables.

---

# Advantages

Insertion Sort provides several advantages:

* Easy to understand and implement.
* Efficient for small datasets.
* Excellent performance on nearly sorted collections.
* Stable sorting algorithm.
* Adaptive to existing order.
* Uses constant extra memory.

---

# Disadvantages

Insertion Sort also has several limitations:

* Slow for large datasets.
* Worst-case performance is quadratic.
* Requires shifting many elements when the collection is sorted in reverse order.
* Less efficient than advanced algorithms such as Merge Sort or Quick Sort.

---

# When Should You Use Insertion Sort?

Insertion Sort is a good choice when:

* The collection is small.
* The data is already mostly sorted.
* Simplicity is preferred.
* Maintaining stability is important.

Typical examples include:

* Small arrays.
* Nearly sorted datasets.
* Educational demonstrations.
* Hybrid sorting algorithms (such as TimSort and IntroSort), where Insertion Sort is often used for sorting small partitions.

# Bubble Sort vs Selection Sort vs Insertion Sort

| Feature          | Bubble Sort             | Selection Sort             | Insertion Sort                              |
| ---------------- | ----------------------- | -------------------------- | ------------------------------------------- |
| Main Idea        | Swap adjacent elements  | Select the minimum element | Insert each element into the sorted portion |
| Best Case        | O(n²)*                  | O(n²)                      | **O(n)**                                    |
| Average Case     | O(n²)                   | O(n²)                      | O(n²)                                       |
| Worst Case       | O(n²)                   | O(n²)                      | O(n²)                                       |
| Space Complexity | O(1)                    | O(1)                       | O(1)                                        |
| Stable           | ✅ Yes                   | ❌ No                       | ✅ Yes                                       |
| Adaptive         | ❌ No*                   | ❌ No                       | ✅ Yes                                       |
| Swaps            | Many                    | One per pass               | Few (mostly shifts)                         |
| Best Use Case    | Learning sorting basics | Minimize swaps             | Small or nearly sorted datasets             |

*An optimized Bubble Sort can achieve **O(n)** in the best case by stopping early when no swaps occur.

---

# Why Is Insertion Sort Important?

Insertion Sort is one of the most practical elementary sorting algorithms.

Although it has quadratic worst-case performance, its excellent behavior on small and nearly sorted datasets makes it an important component of many modern hybrid sorting algorithms.

Understanding Insertion Sort introduces the concept of maintaining a sorted portion of a collection, a technique that appears in many advanced algorithms.
