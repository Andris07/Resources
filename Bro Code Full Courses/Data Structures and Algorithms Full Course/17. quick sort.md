# Quick Sort

# What is Quick Sort?

**Quick Sort** is an efficient comparison-based sorting algorithm that uses the **Divide and Conquer** strategy.

Instead of dividing the collection into equal halves like Merge Sort, Quick Sort selects a **pivot** element and rearranges the collection so that:

* Elements smaller than the pivot are placed on the left.
* Elements greater than the pivot are placed on the right.

The same process is then applied recursively to both partitions until the entire collection is sorted.

Quick Sort is one of the fastest sorting algorithms in practice and is widely used in real-world applications.

---

# Divide and Conquer

Quick Sort follows the **Divide and Conquer** paradigm.

It consists of three main steps:

1. **Choose** a pivot element.
2. **Partition** the collection around the pivot.
3. **Recursively sort** the left and right partitions.

Unlike Merge Sort, Quick Sort performs most of its work during the partitioning phase rather than the merging phase.

---

# Pivot

The **pivot** is the element used to divide the collection.

Different pivot selection strategies exist, including:

* First element
* Last element
* Middle element
* Random element
* Median-of-three

In this implementation, the **last element** is used as the pivot.

A good pivot generally produces two similarly sized partitions, resulting in better performance.

---

# Partition

**Partitioning** rearranges the collection around the pivot.

After partitioning:

* Every element to the left of the pivot is smaller.
* Every element to the right of the pivot is greater.
* The pivot is in its final sorted position.

Example:

```text
Before Partition

8 2 5 3 4 7 6 | 1
                ↑
              Pivot

↓

After Partition

1 | 2 5 3 4 7 6 8
↑
Pivot
```

The algorithm then recursively sorts both partitions.

---

# Pseudocode

```text
QuickSort(array, low, high)

if low >= high

    return

pivot = Partition(...)

QuickSort(left partition)

QuickSort(right partition)
```

# Time Complexity

| Case         | Time Complexity |
| ------------ | --------------- |
| Best Case    | O(n log n)      |
| Average Case | O(n log n)      |
| Worst Case   | O(n²)           |

The worst case occurs when poor pivot choices repeatedly produce highly unbalanced partitions (for example, when sorting an already sorted array while always choosing the first or last element as the pivot).

---

# Space Complexity

| Implementation | Space Complexity |
| -------------- | ---------------- |
| Recursive      | O(log n) average |
| Worst Case     | O(n)             |

Quick Sort is considered an **in-place** sorting algorithm because it rearranges elements within the original collection. However, the recursive function calls still require stack space.

---

# Advantages

Quick Sort provides several advantages:

* Very fast in practice.
* Average time complexity of **O(n log n)**.
* In-place sorting algorithm.
* Excellent cache performance.
* Widely used in production software.

---

# Disadvantages

Quick Sort also has several limitations:

* Worst-case performance is **O(n²)**.
* Performance depends heavily on pivot selection.
* Recursive implementation may cause deep recursion.
* Not a stable sorting algorithm.

---

# When Should You Use Quick Sort?

Quick Sort is an excellent choice when:

* Sorting large arrays.
* Memory usage should remain low.
* Average performance is more important than worst-case guarantees.
* Fast in-place sorting is required.

Typical examples include:

* General-purpose sorting.
* Standard library implementations.
* Competitive programming.
* Large in-memory datasets.

# Merge Sort vs Quick Sort

| Feature                        | Merge Sort         | Quick Sort               |
| ------------------------------ | ------------------ | ------------------------ |
| Strategy                       | Divide, then merge | Partition around a pivot |
| Average Case                   | O(n log n)         | O(n log n)               |
| Worst Case                     | O(n log n)         | O(n²)                    |
| Extra Memory                   | O(n)               | O(log n) average         |
| Stable                         | ✅ Yes              | ❌ No                     |
| In-place                       | ❌ No               | ✅ Yes                    |
| Large Arrays                   | ✅ Excellent        | ✅ Excellent              |
| Linked Lists                   | ✅ Excellent        | ❌ Less suitable          |
| Typical Real-World Performance | Very good          | Excellent                |

---

# Why Is Quick Sort Important?

Quick Sort is one of the most widely used sorting algorithms in computer science.

Its excellent average-case performance, low memory overhead, and divide-and-conquer strategy make it a common choice for sorting large collections. Understanding Quick Sort also introduces key concepts such as pivot selection and partitioning, which are fundamental techniques in algorithm design.
