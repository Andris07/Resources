# Selection Sort

# What is Selection Sort?

**Selection Sort** is a simple comparison-based sorting algorithm.

It repeatedly finds the smallest element in the unsorted portion of the collection and places it in its correct position.

Unlike Bubble Sort, Selection Sort performs **only one swap per pass**, making it more efficient in terms of the number of swaps.

---

# How Does Selection Sort Work?

The algorithm divides the collection into two parts:

* A **sorted** portion at the beginning.
* An **unsorted** portion containing the remaining elements.

During each pass:

1. Find the smallest element in the unsorted portion.
2. Swap it with the first unsorted element.
3. Expand the sorted portion by one element.

Example:

```text
Initial Array

[8] [7] [9] [2] [3] [1]

Smallest = 1

↓

[1] [7] [9] [2] [3] [8]

First position is now sorted.
```

The algorithm repeats this process until every element has been placed in its correct position.

# Visualization

```text
1 2 3 7 9 8

```

Each pass selects the smallest remaining element and places it into its final position.

---

# Pseudocode

```text
SelectionSort(array)

for i from 0 to length - 1

    minimum = i

    for j from i + 1 to length - 1

        if array[j] < array[minimum]

            minimum = j

    swap array[i] and array[minimum]
```

# Time Complexity

| Case         | Time Complexity |
| ------------ | --------------- |
| Best Case    | O(n²)           |
| Average Case | O(n²)           |
| Worst Case   | O(n²)           |

Selection Sort always scans the remaining unsorted elements, regardless of how sorted the collection already is.

---

# Space Complexity

```text
O(1)
```

Selection Sort is an **in-place** sorting algorithm because it requires only a constant amount of additional memory.

---

# Advantages

Selection Sort provides several advantages:

* Easy to understand and implement.
* Performs only one swap per pass.
* Uses constant extra memory.
* Good for educational purposes.
* Performs well when write operations are expensive.

---

# Disadvantages

Selection Sort also has several limitations:

* Slow for large datasets.
* Always performs the same number of comparisons.
* Does not benefit from partially sorted data.
* Much slower than modern sorting algorithms.

---

# When Should You Use Selection Sort?

Selection Sort is useful when:

* Learning sorting algorithms.
* Working with very small datasets.
* Memory usage must remain constant.
* Reducing the number of swaps is important.

Typical examples include:

* Educational demonstrations.
* Embedded systems with limited memory.
* Small collections.

# Bubble Sort vs Selection Sort

| Feature          | Bubble Sort            | Selection Sort              |
| ---------------- | ---------------------- | --------------------------- |
| Main Idea        | Swap adjacent elements | Select the smallest element |
| Swaps            | Many                   | One per pass                |
| Comparisons      | O(n²)                  | O(n²)                       |
| Best Case        | O(n²)*                 | O(n²)                       |
| Average Case     | O(n²)                  | O(n²)                       |
| Worst Case       | O(n²)                  | O(n²)                       |
| Space Complexity | O(1)                   | O(1)                        |
| Stable           | ✅ Yes                  | ❌ No                        |
| Adaptive         | ❌ No*                  | ❌ No                        |

*An optimized Bubble Sort can achieve **O(n)** in the best case by stopping early when no swaps occur.

---

# Why Is Selection Sort Important?

Selection Sort is one of the fundamental sorting algorithms in computer science.

Although it is rarely used in production software, it demonstrates an important algorithmic strategy: repeatedly selecting the best candidate from a set of remaining elements.

Understanding Selection Sort helps build a strong foundation for learning more advanced sorting algorithms such as Heap Sort and Quick Sort.
