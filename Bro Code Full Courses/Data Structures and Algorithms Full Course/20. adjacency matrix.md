# Adjacency Matrix

# What is an Adjacency Matrix?

An **adjacency matrix** is one of the most common ways to represent a graph in memory.

It uses a **two-dimensional array (matrix)** where both rows and columns represent vertices.

Each cell indicates whether an edge exists between two vertices.

Example:

```text
      A  B  C  D

A     0  1  1  0
B     1  0  0  1
C     1  0  0  0
D     0  1  0  0
```

* `1` → Edge exists
* `0` → No edge

---

# How Does It Work?

The graph is stored as a square matrix.

Each position represents a connection between two vertices.

```text
matrix[from][to]
```

Example:

```text
matrix[0][2] = 1
```

means

```text
A → C
```

is connected.

For an **undirected graph**, the matrix is symmetrical.

```text
matrix[A][B] = matrix[B][A]
```

---

# Why Is It Called an Adjacency Matrix?

Two vertices are **adjacent** if they are directly connected by an edge.

The matrix simply records which vertices are adjacent to one another.

# Main Operations

---

## Add Vertex

Adds a new vertex to the graph.

Example:

```python
graph.add_node(Node("A"))
```

---

## Add Edge

Creates a connection between two vertices.

Example:

```python
graph.add_edge(0, 1)
```

Result:

```text
A ------- B
```

---

## Check Edge

Checks whether two vertices are connected.

Example:

```python
graph.check_edge(0, 1)
```

Returns:

```text
True
```

if the edge exists.

---

## Print Matrix

Displays the adjacency matrix.

Example:

```python
graph.print()
```

Output:

```text
  A B C

A 0 1 0
B 1 0 1
C 0 1 0
```

---

# Time Complexity

| Operation              | Complexity |
| ---------------------- | ---------: |
| Add Vertex             |       O(1) |
| Add Edge               |       O(1) |
| Check Edge             |       O(1) |
| Remove Edge            |       O(1) |
| Find Adjacent Vertices |       O(V) |

---

# Space Complexity

```text
O(V²)
```

The matrix contains **V × V** cells, where **V** is the number of vertices.

---

# Advantages

Adjacency matrices provide several advantages:

* Very fast edge lookup.
* Simple implementation.
* Excellent for dense graphs.
* Constant-time edge checking.

---

# Disadvantages

Adjacency matrices also have several limitations:

* High memory usage.
* Inefficient for sparse graphs.
* Adding many vertices may require resizing the matrix.

---

# Adjacency Matrix vs Adjacency List

| Feature           | Adjacency Matrix | Adjacency List |
| ----------------- | ---------------- | -------------- |
| Memory            | O(V²)            | O(V + E)       |
| Edge Lookup       | O(1)             | O(V)           |
| Iterate Neighbors | O(V)             | O(Degree)      |
| Dense Graphs      | ✅ Excellent      | Good           |
| Sparse Graphs     | Poor             | ✅ Excellent    |

---

# Where Are Adjacency Matrices Used?

Common applications include:

* Dense graphs
* Network topology
* Graph algorithms
* Educational implementations
* Small graph datasets

---

# Why Is an Adjacency Matrix Important?

An adjacency matrix provides one of the simplest and fastest ways to represent graphs.

Although it uses more memory than an adjacency list, it allows constant-time edge lookups and serves as the foundation for many graph algorithms studied in computer science.
