# Interpolation Search

# What is Interpolation Search?

**Interpolation Search** is an improved searching algorithm for **sorted and uniformly distributed** collections.

Instead of always checking the middle element like Binary Search, Interpolation Search estimates where the target is likely to be based on its value.

This estimation often allows the algorithm to find the target much faster when the data is evenly distributed.

> **Note:** Interpolation Search only works correctly on **sorted** collections and performs best when the values are **evenly distributed**.

---

# How Does Interpolation Search Work?

Interpolation Search uses the values at the beginning and end of the current search range to estimate the most likely position of the target.

Instead of dividing the search space exactly in half, it calculates a **probe position**.

* If the probe value is smaller than the target, search the right side.
* If the probe value is larger than the target, search the left side.
* Otherwise, the target has been found.

Example:

```text
Evenly Distributed Array

[10] [20] [30] [40] [50] [60] [70] [80] [90]

Searching for 80

Estimated Probe
                                    ↓
[10] [20] [30] [40] [50] [60] [70] [80] [90]

Found!
```

Unlike Binary Search, the probe is not always located in the middle.

---

# Probe Formula

Interpolation Search estimates the probe position using the following formula:

```text
probe = low +
        (high - low)
        x
        (target - array[low])
        /
        (array[high] - array[low])
```

The closer the estimated position is to the target, the fewer comparisons are required.

---

# Pseudocode

```text
InterpolationSearch(array, target)

low = 0
high = array length - 1

while target is within the current range

    estimate probe position

    if array[probe] == target
        return probe

    else if target > array[probe]
        low = probe + 1

    else
        high = probe - 1

return -1
```

| Worst Case    | O(n)            |

*Average-case performance assumes the data is evenly distributed.

# Advantages

Interpolation Search provides several advantages:

* Faster than Binary Search on uniformly distributed data.
* Estimates where the target is likely to be found.
* Requires only constant extra memory.
* Excellent performance for large, evenly distributed datasets.

---

# Disadvantages

Interpolation Search also has several limitations:

* Requires sorted data.
* Performs poorly on unevenly distributed data.
* Worst-case performance is linear.
* More complex than Binary Search.
* Requires numeric values because it relies on arithmetic calculations.

---

# When Should You Use Interpolation Search?

Interpolation Search is a good choice when:

* The collection is sorted.
* Values are approximately evenly distributed.
* The dataset is very large.
* Maximum search performance is desired.

Typical examples include:

* Sequential numeric IDs.
* Student or employee IDs.
* Ordered database records.
* Large numeric datasets.

# Binary Search vs Interpolation Search

| Feature                    | Binary Search       | Interpolation Search           |
| -------------------------- | ------------------- | ------------------------------ |
| Requires Sorted Data       | ✅ Yes               | ✅ Yes                          |
| Requires Even Distribution | ❌ No                | ✅ Yes                          |
| Search Strategy            | Middle element      | Estimated probe position       |
| Best Case                  | O(1)                | O(1)                           |
| Average Case               | O(log n)            | O(log log n)*                  |
| Worst Case                 | O(log n)            | O(n)                           |
| Implementation Complexity  | Easier              | More complex                   |
| Best Use Case              | General sorted data | Large, evenly distributed data |

*When the values are uniformly distributed.

---

# Why Is Interpolation Search Important?

Interpolation Search demonstrates that search algorithms can use information about the **distribution of data**, not just its order.

Although it is more specialized than Binary Search, it can achieve exceptional performance on large, uniformly distributed datasets. Understanding Interpolation Search introduces the idea that choosing the right algorithm often depends on the characteristics of the data being processed.
