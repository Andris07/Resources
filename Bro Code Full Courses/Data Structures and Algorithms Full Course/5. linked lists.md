# Linked List

# What is a Linked List?

A **Linked List** is a linear data structure where elements are stored as individual **nodes**. Unlike an array, the elements are **not stored in contiguous memory**. Instead, each node contains the data itself and one or more references (links) to other nodes.

Because of this structure, Linked Lists can efficiently grow or shrink during runtime without reallocating memory.

---

# LinkedList in Java

In Java, a Linked List is implemented by the `LinkedList` class from the `java.util` package.

The `LinkedList` class implements several interfaces, including:

* **List** – Allows indexed access and list operations.
* **Queue** – Supports FIFO queue operations.
* **Deque** – Supports insertion and removal from both ends of the list.

Because it implements the **Deque** interface, a LinkedList can be used as:

* A List
* A Queue
* A Stack
* A Double-Ended Queue (Deque)

Example:

```java
LinkedList<String> list = new LinkedList<>();
```

---

# Types of Linked Lists

## Singly Linked List

A **Singly Linked List** stores a reference only to the **next** node.

Example:

```text
Head

[A] → [B] → [C] → null
```

Each node contains:

* Data
* Next

---

## Doubly Linked List

A **Doubly Linked List** stores references to both the **next** and the **previous** node.

Example:

```text
null ← [A] ⇄ [B] ⇄ [C] → null
```

Each node contains:

* Data
* Previous
* Next

Java's `LinkedList` is implemented as a **Doubly Linked List**.

---

# Node Structure

Each node in a doubly linked list contains:

```text
Previous ← Data → Next
```

The first node has no previous node.

The last node has no next node.

---

# First and Last Nodes

A LinkedList keeps references to both ends of the list.

* **First** → The first node in the list.
* **Last** → The last node in the list.

Example:

```text
First

↓
[A] ⇄ [B] ⇄ [C]
             ↑

           Last
```

Keeping references to both ends allows very fast insertion and removal at the beginning and the end of the list.

---

# Main Operations

Because LinkedList implements the **Deque** interface, it supports operations on both ends of the list.

---

## Offer First

Adds an element to the beginning of the list.

```java
list.offerFirst("A");
```

---

## Offer Last

Adds an element to the end of the list.

```java
list.offerLast("B");
```

---

## Poll First

Removes and returns the first element.

```java
list.pollFirst();
```

---

## Poll Last

Removes and returns the last element.

```java
list.pollLast();
```

---

## Peek First

Returns the first element without removing it.

```java
list.peekFirst();
```

---

## Peek Last

Returns the last element without removing it.

```java
list.peekLast();
```

---

## Offer

Adds an element to the end of the list.

```java
list.offer("C");
```

Equivalent to:

```java
list.offerLast("C");
```

---

## Poll

Removes and returns the first element.

```java
list.poll();
```

Equivalent to:

```java
list.pollFirst();
```

---

## Add

Adds an element to the end of the list.

```java
list.add("D");
```

---

## Remove

Removes the first occurrence of an element.

```java
list.remove("B");
```

Or remove the first element:

```java
list.remove();
```

---

# Advantages of a Linked List

Linked Lists provide several advantages:

* Dynamic size.
* Fast insertion at the beginning and end.
* Efficient removal of nodes.
* No need for contiguous memory.
* Ideal when the number of elements changes frequently.

---

# Disadvantages of a Linked List

Linked Lists also have several disadvantages:

* Higher memory usage because each node stores references.
* Slower random access than arrays.
* Searching is performed sequentially.
* Poor cache locality compared to arrays.

---

# Time Complexity

| Operation           | Time Complexity |
| ------------------- | --------------- |
| Access by index     | O(n)            |
| Search              | O(n)            |
| Insert at beginning | O(1)            |
| Insert at end       | O(1)            |
| Remove first        | O(1)            |
| Remove last         | O(1)            |
| Insert in middle    | O(n)            |
| Remove in middle    | O(n)            |

---

# Difference Between ArrayList and LinkedList

| ArrayList                        | LinkedList                                      |
| -------------------------------- | ----------------------------------------------- |
| Uses contiguous memory           | Uses linked nodes                               |
| Fast random access               | Slow random access                              |
| Slow insertion/removal in middle | Faster insertion/removal once the node is found |
| Lower memory usage               | Higher memory usage                             |

---

# Where Are Linked Lists Used?

Linked Lists are commonly used when elements are frequently inserted or removed.

Common real-world applications include:

* Implementing **Queues**
* Implementing **Stacks**
* **Deque** implementations
* Music playlists
* Browser navigation (Back / Forward)
* Undo and Redo systems
* Hash table collision handling
* Graph adjacency lists

---

# Why Is a Linked List Important?

A Linked List is one of the fundamental data structures in computer science.

Unlike arrays, Linked Lists prioritize efficient insertion and removal over fast random access. They are widely used as the foundation for more advanced data structures and algorithms, making them an essential topic in Data Structures and Algorithms.
