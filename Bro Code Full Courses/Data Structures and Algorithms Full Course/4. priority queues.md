# Priority Queue

# What is a Priority Queue?

A **Priority Queue** is a special type of queue where elements are removed based on their **priority** rather than strictly following the **FIFO (First In, First Out)** principle.

In a traditional queue, the first element inserted is always the first element removed.

In a Priority Queue, the element with the highest priority is removed first, regardless of when it was inserted.

Example:

```
Offer 30
Offer 10
Offer 20

Priority Queue (Ascending Order):

10 ← Highest Priority
20
30
```

The next element removed would be `10`, even though `30` was inserted first.

---

# Priority Queue in Java

In Java, a Priority Queue is implemented by the `PriorityQueue` class from the `java.util` package.

The `PriorityQueue` class implements the **Queue** interface, meaning it supports the standard queue operations while automatically maintaining a priority order.

Example:

```
PriorityQueue<Integer> queue = new PriorityQueue<>();
```

By default, Java's Priority Queue stores elements in **ascending order**.

---

# FIFO vs Priority-Based Processing

A normal queue follows the FIFO principle:

```
Offer: 30
Offer: 10
Offer: 20

Queue:

Front → 30 10 20 ← Rear

Poll → 30
```

A Priority Queue processes elements according to priority:

```
Offer: 30
Offer: 10
Offer: 20

Priority Queue:

10 20 30

Poll → 10
```

The insertion order is not necessarily preserved.

---

# Main Operations

Because `PriorityQueue` implements the Queue interface, it supports the same common queue operations.

---

## Offer (Insert)

Adds a new element to the queue.

Example:

```
queue.offer(25);
```

After insertion, the queue automatically places the element according to its priority.

---

## Poll (Remove)

Removes and returns the highest-priority element.

Example:

```
queue.poll();
```

In an ascending Priority Queue, this removes the smallest element.

---

## Peek (Examine)

Returns the highest-priority element without removing it.

Example:

```
queue.peek();
```

The queue remains unchanged after the operation.

---

## Element (Examine)

Returns the highest-priority element without removing it.

Example:

```
queue.element();
```

Difference from `peek()`:

* `peek()` returns `null` if the queue is empty.
* `element()` throws an exception if the queue is empty.

---

## Is Empty

Checks whether the queue contains any elements.

Example:

```
queue.isEmpty();
```

---

## Size

Returns the number of elements currently stored.

Example:

```
queue.size();
```

---

# Priority Ordering

The most important feature of a Priority Queue is how priorities are determined.

---

## Ascending Order (Default)

By default, Java uses **natural ordering**.

For numbers, smaller values have higher priority.

Example:

```
PriorityQueue<Integer> queue = new PriorityQueue<>();
```

Elements:

```
30
10
20
```

Removal order:

```
10
20
30
```

---

## Descending Order

A custom comparator can reverse the ordering.

Example:

```
PriorityQueue<Integer> queue =
    new PriorityQueue<>(Collections.reverseOrder());
```

Elements:

```
30
10
20
```

Removal order:

```
30
20
10
```

The largest value now has the highest priority.

---

# Difference Between Queue and Priority Queue

| Feature                   | Queue                  | Priority Queue           |
| ------------------------- | ---------------------- | ------------------------ |
| Processing Order          | FIFO                   | Priority-Based           |
| First Removed Element     | First inserted element | Highest-priority element |
| Insertion Order Preserved | Yes                    | Not necessarily          |
| Queue Interface           | Yes                    | Yes                      |
| Typical Implementation    | LinkedList             | PriorityQueue            |

Example:

```
Inserted:
30, 10, 20
```

Normal Queue:

```
30 → 10 → 20
```

Removal order:

```
30, 10, 20
```

Priority Queue (Ascending):

```
10 → 20 → 30
```

Removal order:

```
10, 20, 30
```

# Advantages of a Priority Queue

* **Automatic ordering** of elements.
* **Efficient access** to the highest-priority element.
* **Fast insertion and removal** operations.
* Useful for scheduling and optimization problems.
* Eliminates the need to sort data after every insertion.

---

# Disadvantages of a Priority Queue

* Does not preserve FIFO ordering.
* Random access is not efficient.
* More complex than a standard queue.
* Maintaining priorities introduces some overhead compared to a simple queue.

---

# Where Are Priority Queues Used?

Priority Queues are commonly used when some tasks or data should be processed before others.

Common real-world applications include:

* **CPU task scheduling** – Higher-priority processes are executed first.
* **Hospital emergency systems** – Critical patients receive treatment before less urgent cases.
* **Network routing algorithms** – Important routes are processed first.
* **Pathfinding algorithms** – Algorithms such as Dijkstra's Algorithm use Priority Queues.
* **Event simulation systems** – Events are processed according to their scheduled priority.
* **Job scheduling systems** – High-priority jobs are executed before lower-priority jobs.

---

# Why Is a Priority Queue Important?

A Priority Queue extends the concept of a traditional queue by allowing elements to be processed according to importance rather than arrival time.

Because many real-world problems involve prioritization, Priority Queues are one of the most useful and frequently used data structures in Data Structures and Algorithms.
