# Stack

# What is a Stack?

A **stack** is a linear data structure that stores elements in a specific order following the **LIFO (Last In, First Out)** principle.

This means that the last element added to the stack is the first element that will be removed.

A stack can be imagined as a stack of plates. You can only add a new plate to the top of the stack, and you can only remove the plate that is currently on the top.

Example:

```
Push 10
Push 20
Push 30

Stack:

30 ← Top
20
10
```

The next value removed from the stack would be `30`.

---

# Main Operations of a Stack

A stack usually supports a small set of fundamental operations that can be found in most programming languages.

---

## Push

The **push** operation adds a new element to the top of the stack.

Example:

```
stack.push(10)
```

After this operation, `10` becomes the new top element.

---

## Pop

The **pop** operation removes the top element from the stack and usually returns its value.

Example:

```
stack.pop()
```

If the stack contains `30, 20, 10`, the operation returns `30` and removes it from the stack.

---

## Peek (or Top)

The **peek** operation returns the element at the top of the stack without removing it.

Example:

```
stack.peek()
```

If the top element is `30`, the stack remains unchanged after the operation.

---

## Is Empty

The **isEmpty** operation checks whether the stack contains any elements.

Example:

```
stack.isEmpty()
```

It usually returns `true` if the stack has no elements and `false` otherwise.

---

## Size

The **size** operation returns the number of elements currently stored in the stack.

Example:

```
stack.size()
```

---

# Advantages of a Stack

Stacks have several benefits:

* **Simple structure** – Easy to understand and implement.
* **Fast operations** – Push and pop operations are usually performed in constant time **O(1)**.
* **Efficient memory usage** – Only the most recently added elements need to be accessed.
* **Useful for tracking previous states** – Allows applications to return to earlier states.

---

# Disadvantages of a Stack

Stacks also have some limitations:

* **Limited access** – Only the top element can be directly accessed.
* **Searching is inefficient** – Finding an element may require checking multiple items.
* **Not suitable for random access** – You cannot instantly access elements in the middle of the stack like in an array.

---

# Where Are Stacks Used?

Stacks are used in many areas of computer science, especially when the order of operations matters.

Common real-world applications include:

* **Function call management** – Programming languages use a call stack to track active functions and return addresses.
* **Undo and redo systems** – Text editors and design applications store previous actions using stacks.
* **Browser history** – The back button can be implemented using a stack of visited pages.
* **Expression evaluation** – Compilers and calculators use stacks to evaluate mathematical expressions.
* **Syntax parsing** – Compilers use stacks to check matching brackets and validate code structures.
* **Depth-First Search (DFS)** – Graph and tree traversal algorithms often rely on a stack.

---

# Why Is a Stack Important?

The stack is one of the most fundamental data structures in computer science.

It provides a simple but powerful way to handle problems where the most recent action or data must be processed first. Because of its efficiency and many practical applications, understanding stacks is an essential part of learning Data Structures and Algorithms.
