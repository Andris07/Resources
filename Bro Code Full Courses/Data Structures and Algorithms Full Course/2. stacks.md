# Stack

# What is a Stack?

A **stack** is a linear data structure that stores elements in a specific order following the **LIFO (Last In, First Out)** principle.

This means that the last element added to the stack is the first element that will be removed.

A stack can be imagined as a stack of plates. You can only add a new plate to the top of the stack, and you can only remove the plate that is currently on the top.

Example:

```
Push 10
Push 20
Push 30

Stack:

Top → 30
       20
       10
```

The next value removed from the stack would be `30`.

---

# Stack in Java

In Java, a stack is most commonly represented by the `Stack` class, which is part of the `java.util` package.

The `Stack` class extends the `Vector` class, meaning it inherits many methods related to dynamic arrays.

Java also provides the `Deque` interface (for example, `ArrayDeque`) as a modern alternative for stack operations because it generally provides better performance and is recommended for new code.

Common Java stack implementations include:

* `Stack` – The traditional stack implementation.
* `ArrayDeque` – The recommended modern implementation of a stack.

---

# Main Operations of a Stack

Most programming languages use the same names for stack operations.

The most common operations are `push`, `pop`, and `peek`.

---

## Push (Insert)

The **push** operation adds a new element to the top of the stack.

In Java, this operation is performed using `push()`.

Example:

```java
stack.push(10);
```

After this operation, `10` becomes the new top element.

---

## Pop (Remove)

The **pop** operation removes the element from the top of the stack and returns its value.

In Java, this operation is performed using `pop()`.

Example:

```java
stack.pop();
```

If the stack contains `30, 20, 10`, the operation returns `30` and removes it from the stack.

---

## Peek (Examine)

The **peek** operation returns the top element without removing it.

In Java, this operation is performed using `peek()`.

Example:

```java
stack.peek();
```

The stack remains unchanged after this operation.

---

## Is Empty

The **isEmpty** operation checks whether the stack contains any elements.

In Java, this operation is performed using `isEmpty()`.

Example:

```java
stack.isEmpty();
```

It returns `true` if the stack has no elements and `false` otherwise.

---

## Search

The **search** operation looks for an element inside the stack.

In Java, `search()` returns the **1-based position** from the top of the stack.

Example:

```java
stack.search(20);
```

If the stack is:

```
Top → 30
       20
       10
```

The result is `2`, because `20` is the second element from the top.

If the element does not exist, the method returns `-1`.

---

# Advantages of a Stack

Stacks have several benefits:

* **Simple structure** – Easy to understand and implement.
* **Fast operations** – Push and pop operations usually have a time complexity of **O(1)**.
* **Efficient memory management** – Only the most recently added elements need to be accessed.
* **Useful for reversing order** – Elements are removed in the opposite order they were inserted.

---

# Disadvantages of a Stack

Stacks also have some limitations:

* **Limited access** – Only the top element can be directly accessed.
* **Searching can be inefficient** – Finding an element may require checking multiple elements.
* **Not suitable for random access** – Elements in the middle cannot be instantly accessed like in an array.

---

# Where Are Stacks Used?

Stacks are used whenever the most recently added information should be processed first.

Common real-world applications include:

* **Function call management** – Programming languages use a call stack to keep track of active function calls.
* **Undo and redo systems** – Text editors and design software store previous actions using stacks.
* **Browser history navigation** – Back and forward operations can be implemented with stacks.
* **Expression evaluation** – Compilers and calculators use stacks to process mathematical expressions.
* **Syntax parsing** – Compilers use stacks to validate brackets, parentheses, and code structures.
* **Depth-First Search (DFS)** – Graph and tree traversal algorithms commonly use stacks.

---

# Why Is a Stack Important?

The stack is one of the fundamental data structures in computer science.

It provides a simple and efficient way to solve problems where the most recently added data must be accessed first. Because of its speed and widespread use in software systems, understanding stacks is an essential part of learning Data Structures and Algorithms.
