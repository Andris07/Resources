# Tree Traversal

# What is Tree Traversal?

**Tree Traversal** is the process of visiting every node in a tree exactly once in a specific order.

Unlike arrays, trees do not store elements sequentially, so traversal algorithms define how the tree should be explored.

The three most common **Depth-First Traversal (DFS)** methods are:

- **Pre-order Traversal**
- **In-order Traversal**
- **Post-order Traversal**

Each traversal method visits the same nodes but in a different order depending on the problem being solved.

Example tree:

```text
            50
          /    \
        30      70
       /  \    /  \
     20   40  60   80
```

---

# Pre-order Traversal

Pre-order traversal visits the **current node first**, then recursively visits the left subtree followed by the right subtree.

Traversal order:

```text
Root → Left → Right
```

Visualization:

```text
            50 (1)
          /        \
      (2)30        70(5)
      /   \       /    \
 (3)20   40(4) 60(6) 80(7)
```

Output:

```text
50
30
20
40
70
60
80
```

Common applications:

- Copying a tree.
- Creating prefix expressions.
- Serializing tree structures.

---

# In-order Traversal

In-order traversal first visits the left subtree, then the current node, and finally the right subtree.

Traversal order:

```text
Left → Root → Right
```

Visualization:

```text
            50 (4)
          /        \
      (2)30        70(6)
      /   \       /    \
 (1)20   40(3) 60(5) 80(7)
```

Output:

```text
20
30
40
50
60
70
80
```

For a **Binary Search Tree (BST)**, in-order traversal always produces the elements in **sorted order**.

Common applications:

- Printing values in ascending order.
- Validating a Binary Search Tree.
- Exporting sorted data.

---

# Post-order Traversal

Post-order traversal first visits the left subtree, then the right subtree, and finally the current node.

Traversal order:

```text
Left → Right → Root
```

Visualization:

```text
            50 (7)
          /        \
      (3)30        70(6)
      /   \       /    \
 (1)20   40(2) 60(4) 80(5)
```

Output:

```text
20
40
30
60
80
70
50
```

Common applications:

- Deleting a tree.
- Releasing dynamically allocated memory.
- Evaluating expression trees.

---

# Traversal Comparison

Using the same tree:

```text
            50
          /    \
        30      70
       /  \    /  \
     20   40  60   80
```

| Traversal | Visit Order |
|-----------|-------------|
| Pre-order | 50 → 30 → 20 → 40 → 70 → 60 → 80 |
| In-order | 20 → 30 → 40 → 50 → 60 → 70 → 80 |
| Post-order | 20 → 40 → 30 → 60 → 80 → 70 → 50 |

# Pseudocode

## Pre-order

```text
Visit current node

Traverse left subtree

Traverse right subtree
```

---

## In-order

```text
Traverse left subtree

Visit current node

Traverse right subtree
```

---

## Post-order

```text
Traverse left subtree

Traverse right subtree

Visit current node
```

---

# Time Complexity

All three traversal algorithms visit every node exactly once.

```text
O(n)
```

Where:

- **n** = Number of nodes

---

# Space Complexity

Average case:

```text
O(log n)
```

Worst case (completely unbalanced tree):

```text
O(n)
```

The additional memory comes from the recursive function call stack.

---

# Advantages

Tree traversal algorithms provide several advantages:

- Visit every node exactly once.
- Simple recursive implementation.
- Efficient for processing hierarchical data.
- Foundation for many advanced tree algorithms.
- Can produce different traversal orders depending on the application.

---

# Disadvantages

Tree traversal also has some limitations:

- Recursive implementations require additional stack memory.
- Deep trees may cause stack overflow.
- Traversing the entire tree can be unnecessary if only one value is needed.

---

# Comparison of Traversal Methods

| Feature | Pre-order | In-order | Post-order |
|---------|-----------|----------|------------|
| Visit Root First | ✅ | ❌ | ❌ |
| Produces Sorted BST Output | ❌ | ✅ | ❌ |
| Root Visited Last | ❌ | ❌ | ✅ |
| Common Use | Copying Trees | Sorted Output | Deleting Trees |

# Where Is Tree Traversal Used?

Tree traversal algorithms are widely used throughout computer science.

Common applications include:

- Binary Search Trees (BSTs)
- Expression Trees
- File system traversal
- XML and HTML document processing
- Abstract Syntax Trees (ASTs)
- Compilers and interpreters
- Database indexing
- Decision trees
- Artificial Intelligence search algorithms

---

# Why Is Tree Traversal Important?

Tree traversal is one of the fundamental concepts of tree-based data structures.

Almost every tree algorithm relies on one of the traversal methods to process nodes in a specific order. Understanding **Pre-order**, **In-order**, and **Post-order** traversal is essential before learning more advanced tree algorithms such as **AVL Trees**, **Red-Black Trees**, **Heaps**, and **Tree Balancing** techniques.
