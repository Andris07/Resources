# Binary Search Tree (BST)

# What is a Binary Search Tree?

A **Binary Search Tree (BST)** is a special type of binary tree in which every node follows a specific ordering rule.

For every node:

- All values in the **left subtree** are smaller.
- All values in the **right subtree** are greater.

This property allows searching, insertion, and deletion to be performed efficiently.

Example:

```text
            50
          /    \
        30      70
       /  \    /  \
     20   40  60   80
```

---

# BST Property

Every node satisfies the following rule:

```text
Left Subtree < Parent < Right Subtree
```

Example:

```text
        50
       /  \
     30    70

30 < 50 < 70
```

Because every subtree is also a BST, this rule applies recursively throughout the tree.

---

# How Does a BST Work?

### Insertion

A new value starts at the root.

- If it is smaller than the current node, move left.
- If it is greater, move right.
- Repeat until an empty position is found.

Example:

Insert **65**

```text
            50
          /    \
        30      70
               /
             60
               \
                65
```

---

### Searching

Searching follows exactly the same rule.

Instead of visiting every node, the search eliminates half of the remaining tree at each step.

Example:

Searching for **60**

```text
50

↓

70

↓

60
```

# Main Operations

## Insert

Adds a new value while preserving the BST property.

Example:

```python
bst.insert(50)
```

---

## Search

Checks whether a value exists.

Example:

```python
bst.contains(40)
```

---

## Find Minimum

Returns the smallest value in the tree.

Example:

```python
bst.find_min()
```

---

## Find Maximum

Returns the largest value in the tree.

Example:

```python
bst.find_max()
```

# Time Complexity

| Operation | Average | Worst |
|-----------|--------:|------:|
| Search | O(log n) | O(n) |
| Insert | O(log n) | O(n) |
| Delete | O(log n) | O(n) |
| Find Min / Max | O(log n) | O(n) |

The worst case occurs when the tree becomes completely unbalanced.

---

# Space Complexity

```text
O(n)
```

where **n** is the number of nodes.

---

# Advantages

Binary Search Trees provide several advantages:

- Fast searching.
- Fast insertion.
- Naturally keeps elements sorted.
- Supports ordered traversal.
- Foundation for many balanced tree structures.

---

# Disadvantages

BSTs also have some limitations:

- Performance decreases if the tree becomes unbalanced.
- More complex than arrays or linked lists.
- Additional memory is required for node references.

---

# Binary Search Tree vs Binary Tree

| Feature | Binary Tree | Binary Search Tree |
|---------|-------------|--------------------|
| Ordering | None | Left < Parent < Right |
| Fast Search | ❌ | ✅ |
| Sorted Traversal | ❌ | ✅ |
| Search Complexity | O(n) | O(log n) average |

---

# Where Are Binary Search Trees Used?

Common applications include:

- Dictionaries.
- Sets.
- Database indexing.
- Symbol tables.
- Autocomplete systems.
- Ordered collections.

---

# Why Is a Binary Search Tree Important?

The Binary Search Tree is one of the most important data structures in computer science.

It combines the hierarchical nature of trees with efficient searching, making it the foundation for many balanced tree structures such as **AVL Trees** and **Red-Black Trees**. Understanding BSTs is essential before learning tree traversal algorithms and self-balancing trees.
