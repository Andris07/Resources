# LinkedList vs ArrayList

# Overview

`ArrayList` and `LinkedList` are two of the most commonly used implementations of the **List** interface in Java.

Although both store ordered collections of elements, they are designed for different use cases and have different internal implementations.

Choosing the right one can significantly affect the performance of your program.

---

# Internal Structure

## ArrayList

An **ArrayList** stores its elements inside a **dynamic array**.

Elements are placed in **contiguous memory**, allowing direct access using an index.

Example:

```text
Index:

0   1   2   3
A | B | C | D
```

---

## LinkedList

A **LinkedList** stores elements as **nodes** connected by references.

Each node stores:

* The data
* A reference to the next node
* A reference to the previous node (Java uses a doubly linked list)

Example:

```text
null ← [A] ⇄ [B] ⇄ [C] ⇄ [D] → null
```

---

# How They Work

## ArrayList

When an element is added, it is stored in the internal array.

If the array becomes full:

1. A larger array is created.
2. All elements are copied.
3. The old array is discarded.
4. The new element is inserted.

Because elements are stored next to each other in memory, accessing an element by index is extremely fast.

---

## LinkedList

Each new element is stored inside its own node.

Nodes are connected using references instead of being placed next to each other in memory.

To access an element, Java must follow the links from one node to another until the desired node is found.

---

# Performance Comparison

| Operation        | ArrayList      | LinkedList |
| ---------------- | -------------- | ---------- |
| Access by index  | O(1)           | O(n)       |
| Search           | O(n)           | O(n)       |
| Add at end       | O(1) amortized | O(1)       |
| Add at beginning | O(n)           | O(1)       |
| Insert in middle | O(n)           | O(n)*      |
| Remove in middle | O(n)           | O(n)*      |
| Memory usage     | Lower          | Higher     |

* Once the node has been located, insertion and removal are **O(1)**. Finding the node still requires **O(n)** time.

---

# Advantages of ArrayList

* Extremely fast random access.
* Better CPU cache performance.
* Lower memory usage.
* Excellent for reading data frequently.
* Usually faster for most applications.

---

# Advantages of LinkedList

* Fast insertion at the beginning.
* Fast removal at the beginning.
* No expensive array resizing.
* Efficient when elements are frequently added or removed from both ends.

---

# Disadvantages of ArrayList

* Slow insertion and removal in the middle.
* Resizing requires copying all elements.
* Inserting at the beginning shifts every element.

---

# Disadvantages of LinkedList

* Slow random access.
* Higher memory consumption because every node stores references.
* Poor cache locality.
* Searching requires traversing the list node by node.

---

# When Should You Use ArrayList?

Choose **ArrayList** when:

* You frequently access elements by index.
* You mostly read data.
* Insertions and removals are relatively rare.
* Performance is important for large collections.

Typical examples:

* User lists
* Product catalogs
* High-score tables
* Configuration data
* Most general-purpose collections

---

# When Should You Use LinkedList?

Choose **LinkedList** when:

* Elements are frequently added or removed at the beginning.
* You need a Queue or Deque implementation.
* Random access is not important.
* The collection changes size very often.

Typical examples:

* Queues
* Deques
* Undo/Redo systems
* Browser navigation
* Task scheduling

---

# Which One Is Better?

Neither data structure is universally better.

* Choose **ArrayList** when **fast access** and overall performance are your priorities.
* Choose **LinkedList** when your application performs **many insertions and removals**, especially at the beginning or the ends of the collection.

In practice, **ArrayList** is the preferred choice for most applications because reading data is much more common than inserting or removing elements in the middle of a collection.

# Summary

| Feature             | ArrayList     | LinkedList                   |
| ------------------- | ------------- | ---------------------------- |
| Internal Structure  | Dynamic Array | Doubly Linked List           |
| Random Access       | ⭐ Excellent   | ❌ Slow                       |
| Insert at Beginning | ❌ Slow        | ⭐ Excellent                  |
| Insert at End       | ⭐ Fast        | ⭐ Fast                       |
| Memory Usage        | ⭐ Lower       | ❌ Higher                     |
| Cache Performance   | ⭐ Better      | ❌ Worse                      |
| Best Use Case       | Reading data  | Frequent insertions/removals |
