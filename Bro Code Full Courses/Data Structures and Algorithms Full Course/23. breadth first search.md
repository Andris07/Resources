# Breadth-First Search (BFS)

# What is Breadth-First Search?

**Breadth-First Search (BFS)** is a graph traversal algorithm that explores a graph **level by level**.

Instead of following one path as deeply as possible, BFS first visits every neighboring vertex before moving to the next level.

To achieve this traversal order, BFS uses a **Queue (FIFO – First In, First Out)**.

---

# How Does BFS Work?

The algorithm follows these general steps:

1. Visit the starting vertex.
2. Mark it as visited.
3. Insert it into a queue.
4. Remove the first vertex from the queue.
5. Visit all of its unvisited neighboring vertices.
6. Mark each neighbor as visited and insert it into the queue.
7. Repeat until the queue becomes empty.

Example graph:

```text
        A
      /   \
     B     C
     |     |
     E     D
```

Possible BFS traversal:

```text
A → B → C → E → D
```

Unlike **Depth-First Search (DFS)**, BFS explores all vertices on the current level before moving deeper into the graph.

# Pseudocode

```text
Create an empty queue

Mark the starting vertex as visited

Enqueue the starting vertex

While the queue is not empty

    Dequeue a vertex

    Process it

    For each adjacent vertex

        If the vertex has not been visited

            Mark it as visited

            Enqueue it
```

---

# Time Complexity

```text
O(V + E)
```

Where:

- **V** = Number of vertices
- **E** = Number of edges

Each vertex and each edge is processed at most once.

---

# Space Complexity

```text
O(V)
```

Additional memory is required for:

- The queue.
- The visited collection.

---

# Advantages

BFS provides several advantages:

- Finds the shortest path in unweighted graphs.
- Visits vertices level by level.
- Easy to understand and implement.
- Commonly used in graph and tree traversal algorithms.

---

# Disadvantages

BFS also has some limitations:

- Requires more memory than DFS.
- Can become expensive for very large graphs.
- Not suitable when deep exploration is preferred.

---

# BFS vs DFS

| Feature | BFS | DFS |
|---------|-----|-----|
| Data Structure | Queue (FIFO) | Stack / Recursion |
| Traversal Style | Level by Level | Deep First |
| Shortest Path | ✅ Yes (Unweighted Graphs) | ❌ No |
| Memory Usage | Usually Higher | Usually Lower |

---

# Where Is BFS Used?

Common real-world applications include:

- Finding the shortest path in unweighted graphs.
- GPS and navigation systems.
- Network broadcasting.
- Social network analysis.
- Web crawlers.
- Level-order traversal of trees.

---

# Why Is Breadth-First Search Important?

Breadth-First Search is one of the most fundamental graph traversal algorithms in computer science.

Its ability to explore graphs level by level makes it the preferred choice whenever the shortest path in an unweighted graph is required. Together with **Depth-First Search (DFS)**, BFS serves as the foundation for many advanced graph algorithms and graph-based applications.
