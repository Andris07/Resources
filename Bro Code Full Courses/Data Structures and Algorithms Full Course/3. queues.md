# Queue

# What is a Queue?

A **queue** is a linear data structure that stores elements in a specific order following the **FIFO (First In, First Out)** principle.

This means that the first element added to the queue is the first element that will be removed.

A queue can be compared to a line of people waiting at a checkout. The first person who joins the line is the first person to be served.

Example:

```
Enqueue 10
Enqueue 20
Enqueue 30

Queue:

Front → 10 20 30 ← Rear
```

The next element removed from the queue would be `10`.

---

# Queue in Java

In Java, a **Queue** is an interface that represents a FIFO collection of elements.

A Queue cannot be instantiated directly. Instead, it is implemented by classes such as:

* **LinkedList** – A general-purpose implementation of the Queue interface.
* **PriorityQueue** – A specialized implementation that processes elements based on priority rather than insertion order.

---

# Main Operations of a Queue

Different programming languages may use different names for queue operations.

The most common names are **enqueue**, **dequeue**, and **peek**. In Java, these operations are usually represented by methods such as `offer`, `poll`, `peek`, and `element`.

---

## Enqueue / Offer (Insert)

The **enqueue** operation adds a new element to the rear of the queue.

In Java, this operation is commonly performed using `offer()`.

Example:

```
queue.offer(10)
```

After this operation, `10` is added to the back of the queue.

---

## Dequeue / Poll (Remove)

The **dequeue** operation removes the element at the front of the queue and usually returns its value.

In Java, this operation is commonly performed using `poll()`.

Example:

```
queue.poll()
```

If the queue contains `10, 20, 30`, the operation returns `10` and removes it from the queue.

---

## Peek / Examine

The **peek** operation returns the front element without removing it.

Example:

```
queue.peek()
```

The queue remains unchanged after this operation.

---

## Element (Examine)

The **element** operation also returns the front element without removing it.

In Java, `element()` behaves similarly to `peek()`, but the main difference is how they handle an empty queue:

* `peek()` returns `null` if the queue is empty.
* `element()` throws an exception if the queue is empty.

Example:

```
queue.element()
```

---

# Advantages of a Queue

Queues provide several benefits:

* **Simple structure** – Easy to understand and implement.
* **Fair processing order** – Elements are processed in the same order they arrive.
* **Fast operations** – Insertion and removal are usually performed in constant time **O(1)**.
* **Useful for scheduling tasks** – Allows operations to be handled in the order they are requested.

---

# Disadvantages of a Queue

Queues also have some limitations:

* **Limited access** – Only the front and rear elements can be directly accessed.
* **Searching can be inefficient** – Finding a specific element may require checking multiple items.
* **No random access** – Elements in the middle cannot be instantly accessed like in an array.

---

# Where Are Queues Used?

Queues are commonly used in systems where tasks must be processed in the order they arrive.

Common real-world applications include:

* **Task scheduling** – Operating systems use queues to manage processes and jobs.
* **Print queues** – Documents are printed in the order they are requested.
* **Message queues** – Applications exchange messages in the order they are sent.
* **Network packet handling** – Incoming packets are often processed using queues.
* **Breadth-First Search (BFS)** – Graph and tree traversal algorithms use queues to visit nodes level by level.

---

# Why Is a Queue Important?

The queue is one of the most fundamental data structures in computer science.

It provides an efficient way to process data in a predictable first-in, first-out order. Because many real-world systems depend on scheduling and ordered processing, understanding queues is an essential part of learning Data Structures and Algorithms.
