# Graph

# What is a Graph?

A **graph** is a non-linear data structure used to represent relationships between objects.

A graph consists of a collection of **vertices (nodes)** connected by **edges**.

Unlike trees, graphs do not require a hierarchical structure. Any vertex can be connected to any other vertex, making graphs ideal for modeling complex relationships.

Examples of real-world graphs include:

* Social networks
* Road maps
* Computer networks
* Flight routes
* Web page links

---

# Graph Terminology

Before working with graphs, it is important to understand a few basic terms.

---

## Vertex (Node)

A **vertex** (also called a **node**) represents an individual object within the graph.

Example:

```text
(A)   (B)   (C)
```

Each letter represents a separate vertex.

---

## Edge

An **edge** represents a connection between two vertices.

Example:

```text
A ------- B
```

The edge indicates that vertex **A** is connected to vertex **B**.

Edges may represent many different relationships, such as:

* Roads between cities
* Friendships
* Internet connections
* Dependencies

---

## Adjacency

Two vertices are considered **adjacent** if they are directly connected by an edge.

Example:

```text
A ------- B ------- C
```

* A is adjacent to B.
* B is adjacent to A and C.
* C is adjacent to B.

A and C are **not** adjacent because no direct edge connects them.

---

# Types of Graphs

---

## Undirected Graph

An **undirected graph** has edges that can be traveled in both directions.

Example:

```text
A ------- B
```

This means:

```text
A ↔ B
```

Both vertices can reach each other.

Common examples include:

* Friendships
* Two-way roads
* Network cables

---

## Directed Graph

A **directed graph** (or **digraph**) contains edges with a specific direction.

Example:

```text
A -------> B
```

This means:

* A can reach B.
* B cannot necessarily reach A.

Common examples include:

* One-way streets
* Website hyperlinks
* Task dependencies

---

# Graph Representations

There are two common ways to store graphs in memory.

---

## Adjacency Matrix

An **adjacency matrix** stores graph connections in a two-dimensional array.

Rows and columns represent vertices, while each cell indicates whether an edge exists.

Example:

```text
      A B C

A     0 1 1
B     1 0 0
C     1 0 0
```

Adjacency matrices provide very fast edge lookups but require more memory.

A separate document covers this representation in detail.

---

## Adjacency List

An **adjacency list** stores, for each vertex, a list of all adjacent vertices.

Example:

```text
A → B → C

B → A

C → A
```

Adjacency lists are much more memory-efficient for sparse graphs.

A separate document covers this representation in detail.

# Advantages of Graphs

Graphs provide several advantages:

* Efficiently represent relationships between objects.
* Can model many real-world systems.
* Flexible structure without hierarchy.
* Support many powerful algorithms.
* Used throughout computer science.

---

# Disadvantages of Graphs

Graphs also have several limitations:

* More complex than linear data structures.
* Some algorithms are difficult to implement.
* Large graphs can consume significant memory.
* Performance depends on the chosen representation.

---

# Where Are Graphs Used?

Graphs are one of the most widely used data structures.

Common applications include:

* Social networks
* GPS and navigation systems
* Computer and communication networks
* Search engines
* Recommendation systems
* Dependency management
* Network routing
* Artificial intelligence
* Game development

---

# Why Are Graphs Important?

Graphs are one of the fundamental data structures in computer science.

They provide a natural way to represent relationships between objects and form the foundation of many important algorithms, including **Depth-First Search (DFS)**, **Breadth-First Search (BFS)**, shortest path algorithms, and minimum spanning tree algorithms.

Understanding graphs is essential for solving many real-world problems involving networks, connectivity, and relationships.
