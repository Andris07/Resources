# Adjacency List

# What is an Adjacency List?

An **adjacency list** is one of the most common ways to represent a graph in memory.

Instead of storing every possible connection like an adjacency matrix, each vertex stores a list containing only its adjacent vertices.

This representation is much more memory-efficient for sparse graphs.

Example:

```text
A → B → C

B → A → E

C → A → D

D → C → E

E → B → D
```

Each row represents a vertex and the vertices directly connected to it.

---

# How Does It Work?

Each vertex owns a collection (usually a list) containing all of its neighboring vertices.

Conceptually, the structure looks like this:

```text
Vertex

↓

List of Adjacent Vertices
```

Example:

```text
A → B → C
```

means that vertex **A** is directly connected to **B** and **C**.

Unlike an adjacency matrix, only existing edges are stored.

---

# Why Is It Called an Adjacency List?

Two vertices are **adjacent** if they are directly connected by an edge.

Each vertex simply stores a list of all adjacent vertices.

# Main Operations

---

## Add Vertex

Adds a new vertex to the graph.

Example:

```python
graph.add_node(Node("A"))
```

---

## Add Edge

Creates a connection between two vertices.

Example:

```python
graph.add_edge(A, B)
```

Result:

```text
A ------- B
```

---

## Check Edge

Checks whether two vertices are connected.

Example:

```python
graph.check_edge(A, B)
```

Returns:

```text
True
```

if an edge exists.

---

## Print Graph

Displays the adjacency list.

Example:

```python
graph.print()
```

Output:

```text
A -> B C
B -> A E
C -> A D
D -> C E
E -> B D
```

---

# Time Complexity

| Operation              | Complexity |
| ---------------------- | ---------: |
| Add Vertex             |       O(1) |
| Add Edge               |       O(1) |
| Check Edge             |       O(V) |
| Remove Edge            |       O(V) |
| Find Adjacent Vertices |  O(Degree) |

`Degree` represents the number of neighbors connected to a particular vertex.

---

# Space Complexity

```text
O(V + E)
```

The adjacency list stores:

* **V** vertices
* **E** edges

Only existing edges consume memory.

---

# Advantages

Adjacency lists provide several advantages:

* Very memory-efficient.
* Ideal for sparse graphs.
* Quickly iterates through neighboring vertices.
* Adding new vertices is simple.
* Stores only existing edges.

---

# Disadvantages

Adjacency lists also have several limitations:

* Edge lookup is slower than with an adjacency matrix.
* Slightly more complex implementation.
* Less efficient for very dense graphs.

---

# Adjacency List vs Adjacency Matrix

| Feature           | Adjacency List | Adjacency Matrix |
| ----------------- | -------------- | ---------------- |
| Memory            | O(V + E)       | O(V²)            |
| Edge Lookup       | O(V)           | O(1)             |
| Iterate Neighbors | O(Degree)      | O(V)             |
| Sparse Graphs     | ✅ Excellent    | Poor             |
| Dense Graphs      | Good           | ✅ Excellent      |

---

# Why Is an Adjacency List Important?

An adjacency list is the most commonly used graph representation in practice.

Because it stores only existing edges, it requires significantly less memory than an adjacency matrix for sparse graphs. Many graph algorithms, including **Depth-First Search (DFS)** and **Breadth-First Search (BFS)**, are commonly implemented using adjacency lists due to their efficiency and flexibility.
