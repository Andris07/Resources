# Binary Search

# What is Binary Search?

**Binary Search** is an efficient searching algorithm that finds a target value in a **sorted collection**.

Instead of checking every element one by one, Binary Search repeatedly divides the search range in half until the target is found or no elements remain.

Because it eliminates half of the remaining elements on each iteration, Binary Search is much faster than Linear Search for large datasets.

> **Note:** Binary Search only works correctly on **sorted** collections.

---

# How Does Binary Search Work?

The algorithm begins by examining the middle element of the collection.

* If the middle element equals the target, the search is complete.
* If the target is smaller, the algorithm continues searching the left half.
* If the target is larger, the algorithm continues searching the right half.
* This process repeats until the target is found or the search range becomes empty.

Example:

```text
Sorted Array:

[10] [20] [30] [40] [50] [60] [70]

                ↑
            Middle = 40

Searching for 60

60 > 40

Discard the left half

[50] [60] [70]

      ↑
  Middle = 60

Found!
```

---

# Pseudocode

```text
BinarySearch(array, target)

low = 0
high = array length - 1

while low <= high

    middle = low + (high - low) / 2

    if array[middle] == target
        return middle

    else if target < array[middle]
        high = middle - 1

    else
        low = middle + 1

return -1
```

# Time Complexity

| Case         | Time Complexity |
| ------------ | --------------- |
| Best Case    | O(1)            |
| Average Case | O(log n)        |
| Worst Case   | O(log n)        |

Each iteration removes approximately half of the remaining search space.

Example:

```text
1,000,000 elements

↓

500,000

↓

250,000

↓

125,000

↓

...
```

Only about **20 comparisons** are needed to search one million sorted elements.

---

# Space Complexity

### Iterative Implementation

```text
O(1)
```

Only a few variables (`low`, `high`, `middle`) are required.

### Recursive Implementation

```text
O(log n)
```

The recursive version requires additional stack memory for recursive function calls.

# Advantages

Binary Search provides several advantages:

* Extremely fast for large datasets.
* Eliminates half of the remaining elements every iteration.
* Requires only constant extra memory when implemented iteratively.
* Much faster than Linear Search on sorted collections.
* Well suited for repeated searches.

---

# Disadvantages

Binary Search also has several limitations:

* Requires the data to be sorted.
* Sorting the data may take additional time.
* Not suitable for linked lists because random access is inefficient.
* More complex than Linear Search.

---

# When Should You Use Binary Search?

Binary Search is an excellent choice when:

* The collection is already sorted.
* Fast searching is required.
* The dataset is large.
* Many searches are performed on the same data.

Typical examples include:

* Searching in sorted arrays.
* Dictionary word lookup.
* Database indexing.
* Finding values in sorted collections.
* Competitive programming.
* Search engines and information retrieval systems.

# Binary Search vs Linear Search

| Feature                     | Linear Search | Binary Search |
| --------------------------- | ------------- | ------------- |
| Requires Sorted Data        | ❌ No          | ✅ Yes         |
| Best Case                   | O(1)          | O(1)          |
| Average Case                | O(n)          | O(log n)      |
| Worst Case                  | O(n)          | O(log n)      |
| Suitable for Large Datasets | ❌ No          | ✅ Yes         |
| Easy to Implement           | ✅ Yes         | ✅ Yes         |

---

# Why Is Binary Search Important?

Binary Search is one of the most fundamental searching algorithms in computer science.

Its logarithmic time complexity makes it dramatically faster than Linear Search for large, sorted datasets. Understanding Binary Search introduces the divide-and-conquer approach, which is used in many advanced algorithms and data structures.
