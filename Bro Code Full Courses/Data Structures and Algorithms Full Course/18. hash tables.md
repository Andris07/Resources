# Hash Tables

# What is a Hash Table?

A **hash table** is a data structure that stores data as **key-value pairs**.

Each key is converted into an array index using a **hash function**, allowing values to be inserted, searched, and removed very efficiently.

Unlike arrays, elements are not stored based on their insertion order. Instead, their storage location depends on the hash value of their key.

Example:

```text
Key         Value

1001  →     Alice
1002  →     Bob
1003  →     Charlie
```

Instead of searching every element one by one, the key is hashed to directly locate the correct bucket.

---

# How Does a Hash Table Work?

A hash table stores its elements inside an array called **buckets**.

When inserting a value:

1. The key's `hashCode()` method is called.
2. The hash code is converted into a bucket index.
3. The key-value pair is stored inside that bucket.

Searching follows exactly the same process.

```text
Key

↓

hashCode()

↓

Hash Value

↓

Bucket Index

↓

Stored Value
```

Because the bucket can usually be calculated immediately, most operations have an average time complexity of **O(1)**.

---

# Hash Code

Every object in Java has a `hashCode()` method.

It returns an integer representing the object.

Example:

```java
Integer number = 25;

System.out.println(number.hashCode());
```

For strings:

```java
String text = "Apple";

System.out.println(text.hashCode());
```

Different objects may produce the same hash code, which is one of the reasons collisions occur.

---

# Capacity

The **capacity** represents the number of buckets inside the hash table.

Example:

```java
final int SIZE = 10;

Hashtable<Integer, String> hashtable = new Hashtable<>(SIZE);
```

or

```java
final int SIZE = 10;

Hashtable<Integer, String> hashtable = new Hashtable<>(SIZE, 0.5f);
```

The second constructor specifies the **load factor**, which determines when the table should automatically resize.

---

# Bucket Index

The bucket is calculated using the modulo (`%`) operator.

Formula:

```text
Bucket = hashCode % Capacity
```

Example:

```text
hashCode = 47

Capacity = 10

47 % 10 = 7

Bucket = 7
```

The modulo operator guarantees that the bucket index remains within the table's capacity.

---

# Collision

A **collision** occurs when two different keys produce the same bucket index.

Example:

```text
Capacity = 10

21 % 10 = 1

31 % 10 = 1

↓

Collision
```

Collisions are completely normal and expected in hash tables.

---

# Chaining

One common way to resolve collisions is called **chaining**.

Each bucket stores a **Linked List** of key-value pairs instead of a single element.

Example:

```text
Bucket 1

↓

[Alice]

↓

[Bob]

↓

[Charlie]
```

When searching for a key, the hash table first locates the bucket, then searches only within that linked list instead of the entire table.

# Main Java Operations

---

## Create a Hash Table

```java
final int SIZE = 10;

Hashtable<Integer, String> hashtable = new Hashtable<>(SIZE);
```

With a custom load factor:

```java
final int SIZE = 10;

Hashtable<Integer, String> hashtable = new Hashtable<>(SIZE, 0.5f);
```

---

## Put (Insert)

Adds a new key-value pair.

```java
hashtable.put(1, "Apple");
```

---

## Get (Retrieve)

Returns the value associated with a key.

```java
hashtable.get(1);
```

---

## Remove

Removes a key-value pair.

```java
hashtable.remove(1);
```

---

## Iterate Through the Table

Integer keys:

```java
final int SIZE = 10;

for (Integer key : hashtable.keySet())
{
    System.out.println(
        key.hashCode() % SIZE +
        "\t" +
        key +
        "\t" +
        hashtable.get(key)
    );
}
```

String keys:

```java
final int SIZE = 10;

for (String key : hashtable.keySet())
{
    System.out.println(
        key.hashCode() +
        "\t" +
        key +
        "\t" +
        hashtable.get(key)
    );
}
```

Display bucket positions:

```java
final int SIZE = 10;

for (String key : hashtable.keySet())
{
    System.out.println(
        key.hashCode() % SIZE +
        "\t" +
        key +
        "\t" +
        hashtable.get(key)
    );
}
```

> **Note:** In modern Java applications, `HashMap` is generally preferred over `Hashtable` because it is not synchronized and usually offers better performance. `Hashtable` is still useful for learning how hash tables work and for maintaining legacy code.

---

# Time Complexity

| Operation | Average | Worst |
| --------- | ------- | ----- |
| Insert    | O(1)    | O(n)  |
| Search    | O(1)    | O(n)  |
| Remove    | O(1)    | O(n)  |

The worst case occurs when many keys collide and end up in the same bucket.

---

# Space Complexity

```text
O(n)
```

A hash table stores every key-value pair and may allocate additional buckets to maintain efficient performance.

---

# Advantages

Hash tables provide several advantages:

* Extremely fast average lookup.
* Fast insertion and removal.
* Stores data using key-value pairs.
* Ideal for implementing dictionaries and maps.
* Average constant-time performance **O(1)**.

---

# Disadvantages

Hash tables also have several limitations:

* Collisions can reduce performance.
* Uses more memory than simple arrays.
* Performance depends on a good hash function.
* Elements are not stored in sorted order.
* Worst-case operations degrade to **O(n)**.

---

# Where Are Hash Tables Used?

Hash tables are widely used throughout computer science.

Common applications include:

* Dictionaries and maps.
* Caches.
* Database indexing.
* Symbol tables in compilers.
* Password lookup.
* Routing tables.
* Duplicate detection.
* Counting frequencies.
* Hash-based collections such as `HashMap` and `Hashtable`.

---

# Why Is a Hash Table Important?

Hash tables are among the most widely used data structures in software development.

Their ability to perform insertion, lookup, and removal in average **O(1)** time makes them an essential building block for modern programming languages, databases, operating systems, and high-performance applications.
