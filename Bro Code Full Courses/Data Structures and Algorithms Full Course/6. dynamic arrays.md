# Dynamic Array (ArrayList)

# What is a Dynamic Array?

A **Dynamic Array** is a data structure that automatically resizes itself as elements are added or removed.

Unlike a regular array, which has a fixed size after creation, a dynamic array can grow (and in some implementations, shrink) during runtime.

The elements are stored in **contiguous memory**, just like a normal array, allowing fast indexed access.

In Java, the most commonly used dynamic array implementation is the `ArrayList` class from the `java.util` package.

Example:

```java
ArrayList<String> list = new ArrayList<>();
```

---

# How Does a Dynamic Array Work?

Internally, an `ArrayList` stores its elements inside a regular array.

For example:

```text
Capacity: 5

Index:

0   1   2   3   4
A | B | C | _ | _
```

The **size** of the list is `3`, while the **capacity** is `5`.

The unused space allows new elements to be inserted without allocating a new array every time.

---

# What Happens "Under the Hood"?

When the internal array becomes full, the ArrayList performs several steps automatically.

Suppose the internal array is full:

```text
Capacity: 4

A | B | C | D
```

You add another element:

```java
list.add("E");
```

Java cannot simply place `"E"` into memory because the array has reached its capacity.

Instead, the following happens internally:

1. A new, larger array is allocated.
2. All existing elements are copied into the new array.
3. The old array becomes eligible for garbage collection.
4. The new element is inserted.
5. The reference now points to the larger array.

Example:

```text
Old Array

A | B | C | D

↓

New Array

A | B | C | D | E | _ | _ | _
```

This resizing operation is relatively expensive, but it happens infrequently, making insertions very efficient on average.

# Main Operations

---

## Add

Adds an element to the end of the list.

Example:

```java
list.add("Apple");
```

Average Time Complexity:

```text
O(1)
```

If resizing is required:

```text
O(n)
```

---

## Add at Index

Inserts an element at a specific position.

Example:

```java
list.add(2, "Banana");
```

All following elements are shifted one position to the right.

Time Complexity:

```text
O(n)
```

---

## Get

Returns the element at a given index.

Example:

```java
list.get(3);
```

Because elements are stored contiguously, this operation is extremely fast.

Time Complexity:

```text
O(1)
```

---

## Set

Replaces the element at a specific index.

Example:

```java
list.set(1, "Orange");
```

Time Complexity:

```text
O(1)
```

---

## Remove

Removes an element.

Example:

```java
list.remove(2);
```

or

```java
list.remove("Apple");
```

Removing an element shifts all following elements to fill the empty position.

Time Complexity:

```text
O(n)
```

---

## Size

Returns the number of stored elements.

Example:

```java
list.size();
```

Time Complexity:

```text
O(1)
```

---

## Clear

Removes all elements.

Example:

```java
list.clear();
```

The internal array may still keep its allocated capacity for future insertions.

---

# Size vs Capacity

These two terms are often confused.

**Size**

The number of elements currently stored.

Example:

```text
[A] [B] [C]

Size = 3
```

---

**Capacity**

The total number of elements the internal array can currently hold before resizing.

Example:

```text
[A] [B] [C] [_] [_]

Capacity = 5
```

Capacity is always greater than or equal to the current size.

---

# Advantages of a Dynamic Array

Dynamic arrays provide several benefits:

* Very fast random access using indexes.
* Automatic resizing.
* Efficient memory usage compared to repeatedly allocating new arrays.
* Easy to use.
* Excellent cache locality because elements are stored contiguously.

---

# Disadvantages of a Dynamic Array

Dynamic arrays also have limitations:

* Inserting or removing elements in the middle requires shifting elements.
* Resizing requires allocating a new array and copying every element.
* Removing many elements does not immediately free memory.
* Insertions at the beginning are inefficient.

---

# Time Complexity

| Operation    | Time Complexity |
| ------------ | --------------- |
| Access       | O(1)            |
| Search       | O(n)            |
| Add (end)    | O(1) amortized  |
| Add (middle) | O(n)            |
| Remove       | O(n)            |
| Set          | O(1)            |
| Resize       | O(n)            |

---

# Dynamic Array vs Linked List

| Dynamic Array (ArrayList)       | LinkedList                                         |
| ------------------------------- | -------------------------------------------------- |
| Contiguous memory               | Linked nodes                                       |
| Fast random access              | Slow random access                                 |
| Insert/remove in middle is slow | Insert/remove after reaching the node is efficient |
| Better cache locality           | Worse cache locality                               |
| Lower memory overhead           | Higher memory overhead                             |

---

# Where Are Dynamic Arrays Used?

Dynamic arrays are one of the most commonly used data structures in software development.

Common real-world applications include:

* Lists of objects
* Collections of users or products
* Game object management
* Database query results
* GUI component collections
* Data processing applications
* Most implementations of the Java Collections Framework

---

# Why Is a Dynamic Array Important?

The Dynamic Array is one of the most fundamental and widely used data structures in computer science.

It combines the fast access of a traditional array with the flexibility of automatic resizing. Because of its excellent performance in most real-world scenarios, the `ArrayList` is often the default choice when an ordered collection of elements is needed.
